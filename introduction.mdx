# Introduction

Splat is a command-line tool that takes the pain out of debugging. Point it at any program entrypoint, and Splat runs your code, captures the error, gathers relevant file context, and returns a structured fix — powered by Groq's fast LLM inference.

## Why Splat?

When a bug hits, you want a fix — not a long debugging session. Splat automates the grunt work: running your code, reading the stack trace, tracing file relationships, and generating a targeted code fix with the exact line to change.

## How Splat works

Splat operates as a pipeline:

<Steps>
  <Step title="Run your entrypoint">
    Splat executes the command you provide (e.g., `python3 main.py`) in a contained subprocess and captures both stdout and stderr.
  </Step>
  <Step title="Parse the error stack">
    The error traceback is parsed to extract every file path mentioned in the stack trace.
  </Step>
  <Step title="Build file context">
    Without `-r`: only files from the error trace are included.  
    With `-r`: an import adjacency graph is built and all Nth-degree related files are included.
  </Step>
  <Step title="Send to Groq LLM">
    The traceback, raw error, and full file context are sent to the Groq API for analysis.
  </Step>
  <Step title="Display structured fix">
    Splat prints the error location, type, description, and suggested code fix in your terminal. You can choose to view and apply the fix interactively.
  </Step>
</Steps>

## Output format

Splat always returns a structured JSON response:

```json
{
  "where": {
    "repository_path": "/path/to/your/project",
    "file_name": "main.py",
    "line_number": "42"
  },
  "what": {
    "error_type": "NameError",
    "description": "Variable 'config' is used before it is defined."
  },
  "how": {
    "error_origination": "42",
    "suggested_code_solution": "config = load_config()"
  }
}
```

## Known limitations

<Warning>
  Splat does not conform to any existing code formatting configurations when inserting suggested fixes. Review the suggestion before applying it.
</Warning>

<Note>
  Splat works best when the entrypoint is run from the project's root directory. Sub-module entrypoints that do not originate in the root directory may not resolve correctly.
</Note>
