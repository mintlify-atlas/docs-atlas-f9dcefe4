Splat is a CLI tool that runs your code, captures errors, gathers full file context, and returns a structured, actionable debug response — all in seconds. Built with Groq's ultra-fast LLM inference, Splat turns cryptic stack traces into clear explanations and suggested fixes.

## Why Splat?

Debugging is slow because it's manual — you read the stack trace, open the relevant files, try to understand how they relate, then search for a fix. Splat automates all of that.

<Steps>
  <Step title="Run your code through Splat">
    Replace `python3 foo.py` with `splat squash "python3 foo.py"`. Splat executes your code in a contained environment and captures the full error output.
  </Step>
  <Step title="Splat builds your error context">
    Splat parses the stack trace to identify every file involved in the error. With the `-r` flag, it recursively follows import relationships to gather Nth-degree related files.
  </Step>
  <Step title="Groq's LLM analyzes the error">
    The error traceback, original error message, and full file context are sent to Groq's `llama3-70b-8192` model. You get back a structured response: where the error is, what it means, and exactly how to fix it.
  </Step>
  <Step title="Apply the fix interactively">
    Splat displays the suggested fix in your terminal and prompts you to apply it directly to your source file — no copy-pasting required.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Groq-powered speed" icon="bolt">
    Uses Groq's hardware-accelerated inference to deliver debug responses nearly instantly, without sacrificing accuracy.
  </Card>
  <Card title="Relational context" icon="diagram-project">
    The `-r` flag traverses your import graph to gather all files related to the error — not just the ones in the stack trace.
  </Card>
  <Card title="Git-aware" icon="code-branch">
    Splat respects your `.gitignore` so sensitive files are never included in the context sent to the LLM.
  </Card>
  <Card title="Interactive fixes" icon="wand-magic-sparkles">
    Review the suggested code change in your terminal and choose whether to apply it to your file with a single keypress.
  </Card>
</CardGroup>
