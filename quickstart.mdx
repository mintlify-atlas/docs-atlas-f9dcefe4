# Quickstart

This guide walks you from installation to your first debug session.

## Prerequisites

- Python 3.8 or later
- pip
- npm (for repomix)
- A [Groq API key](https://console.groq.com)

## Install Splat

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ehcaw/splat.git
    cd splat
    ```
  </Step>
  <Step title="Run setup">
    The setup script creates a virtual environment, installs Python dependencies, and installs repomix via npm.

    ```bash
    bash setup.sh
    ```

    This installs:
    - All Python dependencies from `requirements.txt`
    - `repomix` globally via npm
    - `tmux` via Homebrew (macOS only — on Linux, install tmux manually first)

    <Note>
      The script is written for macOS. If you are on Linux, run `sudo apt install tmux` (or equivalent) before running `setup.sh`.
    </Note>
  </Step>
  <Step title="Activate the virtual environment">
    ```bash
    source zapenv/bin/activate
    ```
  </Step>
  <Step title="Set your Groq API key">
    Create a `.env` file in the project root:

    ```bash .env
    API=your_groq_api_key_here
    ```

    <Note>
      Get your API key from the [Groq Console](https://console.groq.com). The key is read from the `API` environment variable.
    </Note>
  </Step>
  <Step title="Install the splat CLI">
    ```bash
    pip install -e .
    ```
  </Step>
</Steps>

## Run your first debug session

Use `splat squash` followed by the same command you would normally use to run your program:

```bash
splat squash "python3 foo.py"
```

Splat runs your program, captures any error, gathers context from the files in the traceback, and returns a structured fix.

### Example: simple Python error

Suppose you have `foo.py` with a bug:

```python foo.py
def greet(name):
    print(f"Hello, {nme}")

greet("world")
```

Run Splat:

```bash
splat squash "python3 foo.py"
```

Splat outputs something like:

```
🔎 Details about error
✅ We found the first instance of the error at line 2.
✅ The owner of the error: foo.py.
✅ The type of the error: NameError.
✅ Isolated error message: name 'nme' is not defined. Did you mean 'name'?
```

You are then prompted interactively:

```
See suggested change? ... YES / no
```

Select **Yes** (default) to view the suggested fix:

```python
print(f"Hello, {name}")
```

## Use relational analysis for deeper context

When your bug spans multiple files, use the `-r` flag. Splat builds a full import graph from the error trace and includes all related files as context:

```bash
splat squash -r "python3 main.py"
```

<Tip>
  Use `-r` when the error originates in one file but is caused by incorrect values or logic from an imported module. The broader context gives the LLM everything it needs to identify the real root cause.
</Tip>
